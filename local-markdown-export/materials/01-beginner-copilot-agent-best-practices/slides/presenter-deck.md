# Presenter Deck: Beginner Best Practices For GitHub Copilot Agents In VS Code

## Slide 1. Title
On-screen content:
- Beginner Best Practices For GitHub Copilot Agents In VS Code
- Foundation module for BI, DS, and DE teams

Presenter notes:
- This is the behavior baseline for the rest of the curriculum. The focus is not feature coverage. The focus is safe and repeatable use.

## Slide 2. Learning Objectives
On-screen content:
- choose Ask, Plan, or Agent appropriately
- write more specific prompts
- review AI output before accepting it
- understand basic guardrails and limits

Presenter notes:
- Tell learners that these habits matter more than any single shortcut or feature.

## Slide 3. Copilot Is An Assistant, Not An Authority
On-screen content:
- strong drafting tool
- not a replacement for human judgment
- can be wrong, incomplete, or overconfident

Presenter notes:
- Set the tone early: AI output is a draft, not a decision.

## Slide 4. Safe Beginner Workflow
On-screen content:
- Ask for orientation
- Plan for multi-step work
- Agent for bounded execution
- review before accepting

Presenter notes:
- This sequence should become the default mental model for beginners.

## Slide 5. Strong Prompt Basics
On-screen content:
- clear goal
- narrow scope
- intended audience
- expected output
- validation step

Presenter notes:
- This is the easiest reusable template learners can take back to work.

## Slide 6. Give Better Context
On-screen content:
- active file and selection
- `#file`
- `#codebase`
- `#fetch`

Presenter notes:
- Point out that weak context often looks like weak AI, even when the real issue is the prompt setup.

## Slide 7. Common Beginner Mistakes
On-screen content:
- asking for too much at once
- starting with Agent too early
- accepting plausible output without review
- mixing unrelated tasks in one chat

Presenter notes:
- Encourage learners to recognize these as normal mistakes, not personal failures.

## Slide 8. BI Example
On-screen content:
- improve a reporting README safely
- preserve business definitions

Presenter notes:
- Use a reporting or metrics example where language changes could alter meaning.

## Slide 9. DS Example
On-screen content:
- explain a notebook workflow
- ask for reproducibility notes

Presenter notes:
- Connect prompt quality to experimental reliability.

## Slide 10. DE Example
On-screen content:
- review a pipeline README
- ask for validation or troubleshooting guidance

Presenter notes:
- Emphasize operational caution and dependency awareness.

## Slide 11. Review Before Accepting
On-screen content:
- read diffs
- verify commands
- confirm the output matches the ask
- reject or refine when needed

Presenter notes:
- Repeat the rule: never accept a change you cannot explain.

## Slide 12. Hands-On Lab
On-screen content:
- improve one weak prompt
- choose the safest mode
- write one review rule
- define one team guardrail

Presenter notes:
- This is a short lab. The goal is habit formation, not large output.

## Slide 13. Validation Checklist
On-screen content:
- learner can explain modes
- learner can improve prompt quality
- learner can state a review rule

Presenter notes:
- Use this slide to confirm readiness for module 02.

Presenter notes:
