# Slide Outline: Beginner Best Practices For GitHub Copilot Agents In VS Code

## Slide 1. Title
- Beginner Best Practices For GitHub Copilot Agents In VS Code
- Foundation module for BI, DS, and DE teams

## Slide 2. Learning Objectives
- choose Ask, Plan, or Agent appropriately
- write more specific prompts
- review AI output before accepting it
- understand basic guardrails and limits

## Slide 3. Copilot Is An Assistant, Not An Authority
- strong drafting tool
- not a replacement for human judgment
- can be wrong, incomplete, or overconfident

## Slide 4. Safe Beginner Workflow
- Ask for orientation
- Plan for multi-step work
- Agent for bounded execution
- review before accepting

## Slide 5. Strong Prompt Basics
- clear goal
- narrow scope
- intended audience
- expected output
- validation step

## Slide 6. Give Better Context
- active file and selection
- `#file`
- `#codebase`
- `#fetch`

## Slide 7. Common Beginner Mistakes
- asking for too much at once
- starting with Agent too early
- accepting plausible output without review
- mixing unrelated tasks in one chat

## Slide 8. BI Example
- improve a reporting README safely
- preserve business definitions

## Slide 9. DS Example
- explain a notebook workflow
- ask for reproducibility notes

## Slide 10. DE Example
- review a pipeline README
- ask for validation or troubleshooting guidance

## Slide 11. Review Before Accepting
- read diffs
- verify commands
- confirm the output matches the ask
- reject or refine when needed

## Slide 12. Hands-On Lab
- improve one weak prompt
- choose the safest mode
- write one review rule
- define one team guardrail

## Slide 13. Validation Checklist
- learner can explain modes
- learner can improve prompt quality
- learner can state a review rule

## Slide 14. Reflection
- what guardrail matters most?
- what mistake is most likely in your team?
