OBJECTIVES 2/15

Learning Objectives

  • choose Ask, Plan, or Agent appropriately
  • write more specific prompts
  • review AI output before accepting it
  • understand basic guardrails and limits
Tell learners that these habits matter more than any single shortcut or feature.
3/15

Copilot Is An Assistant, Not An Authority

  • strong drafting tool
  • not a replacement for human judgment
  • can be wrong, incomplete, or overconfident
Set the tone early: AI output is a draft, not a decision.
4/15

Safe Beginner Workflow

  • Ask for orientation
  • Plan for multi-step work
  • Agent for bounded execution
  • review before accepting
This sequence should become the default mental model for beginners.
5/15

Strong Prompt Basics

  • clear goal
  • narrow scope
  • intended audience
  • expected output
  • validation step
This is the easiest reusable template learners can take back to work.
6/15

Give Better Context

  • active file and selection
  • `#file`
  • `#codebase`
  • `#fetch`
Point out that weak context often looks like weak AI, even when the real issue is the prompt setup.
7/15

Common Beginner Mistakes

  • asking for too much at once
  • starting with Agent too early
  • accepting plausible output without review
  • mixing unrelated tasks in one chat
Encourage learners to recognize these as normal mistakes, not personal failures.
EXAMPLE 8/15

BI Example

  • improve a sales KPI glossary safely
  • preserve revenue, margin, and YTD definitions
Use a concrete example such as a sales dashboard glossary where terms like gross margin and net sales already have approved definitions.
EXAMPLE 9/15

DS Example

  • explain a churn-model notebook workflow
  • ask for environment, seed, and data-version notes
Connect prompt quality to experimental reliability. Show how a missing seed or data snapshot note can make a notebook look reproducible when it is not.
EXAMPLE 10/15

DE Example

  • review an Airflow pipeline runbook
  • ask for upstream dependency and retry validation guidance
Emphasize operational caution and dependency awareness. A safe improvement is clarifying what to check before rerunning a failed daily load.
11/15

Review Before Accepting

  • read diffs
  • verify commands
  • confirm the output matches the ask
  • reject or refine when needed
Repeat the rule: never accept a change you cannot explain.
