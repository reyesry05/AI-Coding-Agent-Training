# Slide Outline: Advanced Skills, Agents, And Plugins

## Slide 1. Module Purpose

- Position module 08 as the advanced follow-on to the beginner sequence.
- Explain that this module teaches selection, composition, and governance.

## Slide 2. Why This Matters

- Teams often confuse reusable guidance, execution workflows, and integrations.
- Misclassification creates unnecessary complexity and weak reviewability.

## Slide 3. Core Definitions

- Skill: reusable guidance for a type of work.
- Agent: task-oriented execution workflow.
- Plugin: external capability or integration layer.

## Slide 3b. Real Examples From GitHub Copilot

- Coding Agent (GA): assigns issues to Copilot for autonomous implementation
- Code Review (GA): automated PR reviewer
- Copilot Spaces (GA): shared knowledge context (skill-like)
- MCP Servers (GA): external tool access (plugin)
- Agent HQ (Preview): central agent dashboard
- Custom Agents (Preview): user-defined agents
- Metrics Dashboard (GA): org-level usage analytics (governance)

## Slide 4. Decision Rules

- Use a skill for repeatable method.
- Use an agent for multi-step task completion.
- Use a plugin for system or tool access.

## Slide 5. How They Work Together

- Skill defines the method.
- Agent executes the method.
- Plugin provides capability access.
